# üß© **JENKINS ‚Äì INTERVIEW NOTES (For Interns / Freshers)**

---

## üîπ 1Ô∏è‚É£ What is Jenkins?

* Jenkins is an **Open Source Automation Server**.
* Used for **Continuous Integration (CI)** and **Continuous Deployment (CD)**.
* Developed in **Java**.
* Automates: Build ‚Üí Test ‚Üí Deploy ‚Üí Monitor.

üí¨ Example:
Whenever a developer pushes code to GitHub ‚Üí Jenkins automatically builds and tests it.

---

## üîπ 2Ô∏è‚É£ Why Jenkins is Used?

* To automate the **software development lifecycle (SDLC)**.
* Reduces manual work.
* Detects bugs early through continuous testing.
* Supports integration with 1000+ tools (Git, Docker, AWS, etc.).

---

## üîπ 3Ô∏è‚É£ Jenkins Architecture

**Components:**

1. **Jenkins Master (Controller):**

   * Controls jobs and schedules builds.
2. **Jenkins Agent (Slave):**

   * Executes the jobs assigned by master.
3. **Jenkins Dashboard:**

   * Web UI for creating and managing jobs.

üí° Master-Node concept allows **distributed builds**.

---

## üîπ 4Ô∏è‚É£ Common Jenkins Terms

| Term      | Meaning                                          |
| --------- | ------------------------------------------------ |
| Job       | Task or project you create in Jenkins            |
| Build     | Execution of that job                            |
| Node      | Machine (master or agent) running Jenkins        |
| Executor  | Thread that runs a build                         |
| Workspace | Folder where Jenkins stores files while building |
| Pipeline  | Automated flow (Build ‚Üí Test ‚Üí Deploy)           |

---

## üîπ 5Ô∏è‚É£ Jenkins Job Types

| Type                             | Use                                      |
| -------------------------------- | ---------------------------------------- |
| **Freestyle Project**            | Simple, manual build steps               |
| **Pipeline Project**             | Scripted automation using Jenkinsfile    |
| **Multibranch Pipeline**         | Auto build for each Git branch           |
| **Multi-Configuration (Matrix)** | Test same code on different environments |
| **Folder**                       | Organize multiple jobs                   |
| **Maven Project**                | For Java + Maven builds                  |
| **External Job**                 | Track external system process            |

---

## üîπ 6Ô∏è‚É£ Jenkins Pipeline

* Written using **Groovy syntax** in a file called `Jenkinsfile`.
* Defines stages like **Build**, **Test**, **Deploy**.

**Example:**

```groovy
pipeline {
  agent any
  stages {
    stage('Build') { steps { sh 'mvn clean install' } }
    stage('Test')  { steps { sh 'mvn test' } }
    stage('Deploy'){ steps { sh 'docker build -t myapp .' } }
  }
}
```

üí° **Two Types of Pipelines:**

1. **Declarative Pipeline** ‚Üí easy syntax
2. **Scripted Pipeline** ‚Üí full Groovy scripting flexibility

---

## üîπ 7Ô∏è‚É£ Jenkinsfile

* File that stores pipeline code.
* Kept in Git repository (with the source code).
* Helps in **Pipeline as Code** concept.

---

## üîπ 8Ô∏è‚É£ Plugins in Jenkins

* Plugins extend Jenkins functionality.
* Common plugins:

  * **Git Plugin** ‚Äì for Git integration
  * **Pipeline Plugin** ‚Äì for pipeline jobs
  * **Docker Plugin** ‚Äì to build/deploy Docker containers
  * **Blue Ocean** ‚Äì modern UI for pipelines

## üîπ 9Ô∏è‚É£ Continuous Integration / Deployment Flow

1. Developer pushes code to GitHub
2. Jenkins triggers automatically (via Webhook)
3. Build ‚Üí Test ‚Üí Deploy steps run
4. Jenkins gives success/failure report

üí° **This is CI/CD process.**

 Difference Between CI, CD (Delivery) and CD (Deployment)
Point	CI (Continuous Integration)	CD (Continuous Delivery)	CD (Continuous Deployment)
üîπ Meaning	Code ko frequently merge karke automatically build & test karna	Code ko release ke liye automatically ready karna	Code ko production (live) par automatically deploy karna
‚öôÔ∏è Main Work	Build and Test	Prepare for Release	Deploy to Production

üéØ Goal	Detect bugs early	Always keep code ready for release	Deliver code to users quickly
üß© Example	Developer pushes code ‚Üí Jenkins build & test karta hai	Jenkins deploy-ready package banata hai	Jenkins code ko live website par deploy karta hai
üïí When Used	During code development	After testing but before production	After final approval (or directly if no approval)
üîç Easy Way to Remember

üß† CD (Delivery) = Code ready forsss deployment

### üß© **1. What is `agent.jar`?**

`agent.jar` ek **Jenkins agent connector file** hai (Java file hoti hai).
Ye ek **bridge (connection)** banati hai **Jenkins master (controller)** aur **slave (agent machine)** ke beech.

---

### ‚öôÔ∏è **2. Why we use `agent.jar`?**

Hum Jenkins me multiple nodes (machines) use karte hain workload divide karne ke liye.
Master ko ye batana padta hai ki slave machine se kaise connect ho.
Ye connection establish karta hai `agent.jar` file.

Basically:

> **agent.jar helps master and agent communicate** ‚Äî send jobs, logs, files, etc.

---

### ‚è∞ **3. When we use it?**

Jab hum **slave (agent node)** setup karte hain manually (EC2 instance, remote machine, etc.)
tab Jenkins ke dashboard me agent create karne ke baad Jenkins ek **command deta hai** jisme ye hota hai üëá

```bash
curl -sO http://<JENKINS_IP>:8080/jnlpJars/agent.jar
java -jar agent.jar -url http://<JENKINS_IP>:8080/ -secret <SECRET_KEY> -name "<NODE_NAME>" -webSocket -workDir "/home/ubuntu"
```

Ye command slave machine pe run karte hain.
Isse slave machine pe ek **JAR file download hoti hai (`agent.jar`)** aur master ke sath connection ban jata hai ‚úÖ

---

### üí° **4. Summary in short:**

| Step          | Description                           |
| ------------- | ------------------------------------- |
| **File name** | agent.jar                             |
| **Location**  | Downloaded on agent machine           |
| **Purpose**   | To connect Jenkins master and agent   |
| **When used** | During agent/slave node setup         |
| **Language**  | Java (it‚Äôs a JAR = Java Archive file) |
